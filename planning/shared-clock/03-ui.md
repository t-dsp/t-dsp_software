# 03 — UI

## Clock tab

New top-level tab in the web dev surface, registered in
[main.ts](../../projects/t-dsp_tac5212_audio_shield_adaptor/tools/web_dev_surface/src/main.ts)
alongside Mixer / Spectrum / Synth / FX / Processing / Loop / Beats.
Panel lives at
[ui/clock-panel.ts](../../projects/t-dsp_tac5212_audio_shield_adaptor/tools/web_dev_surface/src/ui/clock-panel.ts).

## Layout

Four modules stacked in the standard `.processing-panel` column:

1. **Status** — 48px BPM readout (rounds to integer when steady,
   one decimal otherwise) + transport LED (green when running) +
   "RUNNING" / "STOPPED" label.
2. **Source** — two side-by-side cards:
   - **External** — "Slave to MIDI clock"
   - **Internal** — "This device is master"
   Active card gets the `.active` accent.
3. **Tempo** — BPM slider (20..300, step 0.1) + ±1 / ±5 nudge
   buttons. Disabled in External mode (the upstream is authoritative;
   the slider still displays the measured tempo).
4. **Meter** — `<input type="number">` for beats-per-bar (1..16).

Plus a `.proc-note` paragraph explaining what "one shared clock"
means for downstream consumers.

## State mirror

```ts
export interface ClockState {
  source:       Signal<'ext' | 'int'>;
  bpm:          Signal<number>;   // 20..300
  running:      Signal<boolean>;
  beatsPerBar:  Signal<number>;   // 1..16
}
```

Same Signal pattern as every other panel — optimistic local update on
user gesture, OSC write, firmware echo lands back in
`Dispatcher.handleIncoming()` and re-applies (idempotent).

Looper state also picked up two new signals:

```ts
quantize:  Signal<boolean>;
armed:     Signal<number>;   // 0..4 per the /loop/armed encoding
```

## Dispatcher surface

```ts
// Clock
setClockSource(s: 'ext' | 'int'): void
setClockBpm(v: number): void         // throttled ~30 Hz
setClockBeatsPerBar(n: number): void
queryClockRunning(): void             // one-shot probe for /clock/running

// Loop (new)
setLooperQuantize(on: boolean): void
```

Inbound handlers cover `/clock/source|bpm|running|beatsPerBar` and
`/loop/quantize|armed`. All clamp or coerce malformed values rather
than propagating them to Signals.

## Styles

New classes in [style.css](../../projects/t-dsp_tac5212_audio_shield_adaptor/tools/web_dev_surface/src/style.css):

- `.clock-bpm-big` / `.clock-bpm-suffix` / `.clock-bpm-block` —
  the big readout.
- `.clock-led` / `.clock-led.on` — transport LED with
  `box-shadow` glow when running.
- `.clock-source-bar` / `.clock-source-btn` / `.clock-source-label` /
  `.clock-source-sub` — the two-card source picker.
- `.clock-nudge` / `.clock-nudge-btn` — ± tempo buttons, with
  `:disabled` styling for External mode.
- `.clock-bpb` — small numeric input for beats-per-bar.

Existing `.processing-panel`, `.proc-module`, `.proc-row`,
`.proc-readout`, `.proc-note` are reused for structure.

## Tab lifecycle

No subscription setup / teardown needed — clock echoes are
change-driven, not streaming. The panel fires
`dispatcher.queryClockRunning()` in a `queueMicrotask` on mount so
the transport LED is correct the first time the tab is opened;
nothing needs to happen on leave.
